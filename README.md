# 💬 AI-Powered Customer Support Chatbot

An intelligent, FAQ-based customer support chatbot built with **FastAPI**, **Streamlit**, **NLTK**, and **Sentence-Transformers**. The chatbot uses semantic search (cosine similarity over sentence embeddings) to understand customer questions in natural language and return the most relevant answer from a curated knowledge base — falling back gracefully when it isn't confident in a match. Every interaction is timestamped and logged to a local SQLite database, with built-in search over past conversations.

## ✨ Features

- 🤖 **Semantic FAQ Matching** — uses `Sentence-Transformers (all-MiniLM-L6-v2)` and cosine similarity to find the best-matching FAQ answer, not just keyword matching.
- 💬 **ChatGPT-style UI** — clean, modern chat bubbles built in Streamlit, with distinct styling for user vs. bot messages.
- 🕒 **Timestamps** on every message.
- 🧠 **Context-aware session** — the engine keeps a rolling window of recent turns for future contextual extensions.
- 🗄️ **SQLite logging** — every user message + bot response is saved with a timestamp.
- 🔍 **Conversation search** — search past conversations by keyword directly from the sidebar or via the API.
- 🧹 **Clear Chat** button to reset the current session.
- ⚙️ **FastAPI backend** with interactive Swagger docs at `/docs`.
- ✅ **Input validation & error handling** throughout (empty messages, oversized input, model/DB failures).
- 🧩 **Modular, OOP-based architecture** — `ChatbotEngine`, `FAQKnowledgeBase`, `TextPreprocessor`, and `ChatDatabase` are all independent, testable classes.
- ☁️ **Deployable on Streamlit Cloud** with zero extra infrastructure (the chatbot logic runs in-process — no separate API server required).

## ⚙️ How It Works

1. **FAQ Knowledge Base** (`faq.json`) contains structured Q&A pairs across categories like *Account, Billing, Orders, Technical, Support,* and *General*.
2. On startup, `chatbot.py` loads the FAQ file and encodes every question into a dense vector embedding using the `all-MiniLM-L6-v2` Sentence-Transformer model. These embeddings are computed **once** and cached in memory.
3. When a user sends a message:
   - Simple intents (greetings, thanks, goodbye) are detected first via lightweight rule-based matching for instant, natural responses.
   - Otherwise, the message is encoded into an embedding and compared against all FAQ embeddings using **cosine similarity**.
   - The FAQ with the highest similarity score is selected.
   - If the similarity score is **below the confidence threshold (default `0.45`)**, the bot replies: *"Sorry, I couldn't understand your question."*
4. Every exchange (user message, bot response, timestamp) is persisted to the `chat_logs` table in SQLite.
5. The Streamlit UI renders the conversation as chat bubbles, with timestamps and (optionally) confidence scores, and lets users search or clear history.

## 💻 Installation

### Prerequisites
- Python 3.10 or higher
- pip
- (Recommended) a virtual environment tool such as `venv` or `conda`

### Steps

```bash
# 1. Clone or download the project
git clone <your-repo-url> AI_Chatbot
cd AI_Chatbot

# 2. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. (First run only) NLTK data and the Sentence-Transformer model will be
#    downloaded automatically the first time chatbot.py runs. This requires
#    an internet connection on first launch.
```

> 💡 The first time you run the app, the `all-MiniLM-L6-v2` model (~90MB) will be downloaded automatically and cached inside the `models/` directory. Subsequent runs will load it from the local cache and will not need internet access.

## 🚀 Running the App

### Option 1 — Streamlit only (recommended, simplest)

The Streamlit app imports the chatbot engine and database directly — **no separate API server needed**. This is the easiest way to run the project locally or deploy it on Streamlit Cloud.

```bash
streamlit run app.py
```

Then open the URL shown in your terminal (typically `http://localhost:8501`).

### Option 2 — Run FastAPI backend + Streamlit frontend separately

If you want to expose a REST API (e.g., for mobile apps, other clients, or microservice-style architecture):

**Terminal 1 — start the API:**
```bash
uvicorn api:app --reload --host 0.0.0.0 --port 8000
```
Swagger docs will be available at: `http://localhost:8000/docs`

**Terminal 2 — start Streamlit in API mode:**
```bash
# Tell the Streamlit app to call the API instead of running the model in-process
export CHATBOT_BACKEND_MODE=api          # Windows (PowerShell): $env:CHATBOT_BACKEND_MODE="api"
export CHATBOT_API_URL=http://localhost:8000
streamlit run app.py
```

### Key Endpoints

#### `POST /chat`
Send a user message and receive a chatbot response.

**Request body:**
```json
{
  "message": "How can I reset my password?"
}
```

**Response:**
```json
{
  "response": "Click on the 'Forgot Password' option on the login page, enter your registered email address, and follow the instructions sent to your inbox to create a new password.",
  "confidence": 0.8821,
  "matched_question": "How can I reset my password?",
  "category": "Account"
}
```

#### `GET /history?limit=50`
Returns the full (or limited) conversation history stored in SQLite.

#### `GET /search?keyword=refund`
Searches past conversations for a keyword in either the user message or bot response.

#### `DELETE /history`
Clears all stored conversation history.

#### `GET /health`
Simple health check endpoint (useful for uptime monitors / load balancers).

## 🗄 Database Schema

SQLite database file: `data/chatbot.db`

### Table: `chat_logs`

| Column         | Type    | Description                          |
|-----------------|----------|----------------------------------------|
| `id`            | INTEGER  | Primary key, auto-incremented           |
| `user_message`  | TEXT     | The message sent by the user            |
| `bot_response`  | TEXT     | The chatbot's generated reply           |
| `timestamp`     | TEXT     | `YYYY-MM-DD HH:MM:SS` of the interaction |

You can inspect the database manually with any SQLite client:

```bash
sqlite3 data/chatbot.db "SELECT * FROM chat_logs ORDER BY id DESC LIMIT 10;"
```

## 🔧 Configuration

The following can be adjusted directly in code or via environment variables:

| Setting                     | Location                          | Default              |
|------------------------------|-------------------------------------|------------------------|
| Confidence threshold          | `chatbot.py` → `CONFIDENCE_THRESHOLD` | `0.45`                |
| Embedding model               | `chatbot.py` → `MODEL_NAME`         | `all-MiniLM-L6-v2`    |
| Fallback response text         | `chatbot.py` → `FALLBACK_RESPONSE`  | *"Sorry, I couldn't understand your question."* |
| Backend mode (Streamlit)       | env var `CHATBOT_BACKEND_MODE`     | `direct`              |
| API base URL (Streamlit)       | env var `CHATBOT_API_URL`           | `http://localhost:8000` |
| FAQ knowledge base              | `faq.json`                          | 24 sample Q&A pairs    |

To customize the chatbot's knowledge, simply edit `faq.json` — each entry requires `id`, `category`, `question`, and `answer` fields. No retraining is required; embeddings are recomputed automatically on the next app restart.

## ☁️ Deployment on Streamlit Cloud

This project is designed to deploy on **Streamlit Community Cloud** with minimal configuration, since `app.py` runs the chatbot engine in-process (`CHATBOT_BACKEND_MODE=direct` by default) — no separate backend server is required.

### Steps:

1. **Push your project to a public (or private) GitHub repository.**
   ```bash
   git init
   git add .
   git commit -m "Initial commit: AI Customer Support Chatbot"
   git branch -M main
   git remote add origin <your-repo-url>
   git push -u origin main
   ```

2. **Go to [Streamlit Community Cloud](https://streamlit.io/cloud)** and sign in with GitHub.

3. Click **"New app"**, then select:
   - Repository: your repo
   - Branch: `main`
   - Main file path: `app.py`

4. Click **"Deploy"**.

5. On first launch, Streamlit Cloud will install dependencies from `requirements.txt` and download the Sentence-Transformer model and NLTK data automatically (this can take a couple of minutes on the very first boot).

6. Your chatbot will be live at a URL like:
   ```
   https://<your-app-name>.streamlit.app
   ```

### Notes for Cloud Deployment
- SQLite storage on Streamlit Cloud is **ephemeral** — the filesystem resets on redeploys/restarts. For persistent production logging, consider swapping `database.py`'s SQLite connection for a hosted database (e.g., PostgreSQL, Supabase, or Turso) while keeping the same `ChatDatabase` interface.
- The `models/` cache folder also resets on redeploy, so the embedding model will re-download on the next cold start — this is expected and only adds a short delay.
- If you also want to deploy the FastAPI backend (`api.py`) separately (e.g., on Render, Railway, or Fly.io), update `CHATBOT_API_URL` and set `CHATBOT_BACKEND_MODE=api` in your Streamlit Cloud app's secrets/environment settings.

## 🐞 Troubleshooting

| Issue                                                     | Solution                                                                 |
|---------------------------------------------------------------|------------------------------------------------------------------------------|
| `ModuleNotFoundError` on first run                              | Run `pip install -r requirements.txt` inside your active virtual environment. |
| NLTK resource errors (`punkt`, `stopwords`, etc.)                 | Ensure you have internet access on first run; resources auto-download. You can also manually run `python -m nltk.downloader punkt punkt_tab stopwords wordnet omw-1.4`. |
| Sentence-Transformer model fails to download                     | Check internet connectivity; the model is cached in `models/` after the first successful download. |
| `sqlite3.OperationalError: database is locked`                    | Avoid running multiple write-heavy processes against the same `data/chatbot.db` simultaneously. |
| Streamlit shows stale chat after clearing                          | Click "Clear Chat" again or refresh the browser tab — `st.rerun()` should handle this automatically. |
| Bot always returns the fallback message                            | Lower `CONFIDENCE_THRESHOLD` in `chatbot.py`, or check that `faq.json` loaded correctly (no JSON syntax errors). |
